Installation

Drop one CSS file into your project and you're done — that's it, seriously! No complicated build tools, no terminal commands, no package managers required. Just download grit.css (or link to it from a CDN) and add a single <link> tag to your HTML. It's like buying a coloring book that comes with the crayons already inside.

Usage

Once you've linked the stylesheet, just write normal HTML and everything will look polished right away. Most standard HTML elements — headings, paragraphs, buttons, inputs, tables — get styled automatically without you adding any CSS classes at all. Think of it like a magic paintbrush: you draw the shapes (HTML), and grit.css colors them in beautifully for you.

Dark Mode

grit.css has built-in dark mode that works automatically! If your computer or phone is set to dark mode, the page will follow along — no setup needed. But if you want to add a manual toggle (like a light switch for your website), just add or remove the .dark class on the <html> element. You can also force light mode with a .light class if you want to override the system setting.

Themes

Themes let you re-skin any part of your page by adding a single class to a container. Everything inside that container — buttons, badges, progress bars, you name it — will automatically adopt the theme's color palette. It's like putting on tinted sunglasses: everything you see through them changes color, but the world underneath stays the same. grit.css ships with three built-in themes: theme-midnight (deep blue), theme-forest (earthy green), and theme-rose (warm pink).

Navbar

The navigation bar that sits at the top of your page. It's like the menu at a restaurant — it tells people where they can go. On big screens it shows all the links in a row, and on phones it shrinks down to a little hamburger button (☰) that opens a dropdown menu when you tap it. It uses the .navbar class on a <nav> element, a .brand link for your site name, a .navbar-toggle button for the hamburger icon, and a .navbar-menu wrapper around all the links. The hamburger animation (three lines morphing into an X) is handled entirely by CSS — the only JavaScript you need is a tiny snippet that toggles the .open class on the menu and flips aria-expanded on the button.

Buttons

Buttons are the clickable things on your page. grit.css gives you a rainbow of color variants (primary, success, error, etc.), different visual styles (solid, outline, ghost), and size options (small, large, pill-shaped). Mix and match classes freely — just start with .btn and stack on whatever you want. You can also make them full-width with .btn-block or dim them out with the disabled attribute.

Cards

Cards are like little containers with a border and some padding — imagine index cards on a desk. They're perfect for grouping related content together, like a blog post preview or a product listing. You can add images on top, a title, some text, and action buttons at the bottom. The .card class is the wrapper, .card-body adds the padding, .card-title styles the heading, and .card-actions lines up the buttons at the bottom.

Badges

Badges are those tiny little labels you see next to things — like the red circle on your phone's app icons that says you have 3 new messages. They come in every color variant, can be solid or outline, and have a larger .badge-lg size if you need them to stand out more. Use them to show statuses, counts, or categories.

Alerts

Alerts are colored boxes that grab your attention — they're how your page says "hey, look at this!" to your users. Each alert type has a different color and a thick left border: alert-info (blue, for general info), alert-success (green, for good news), alert-warning (yellow, for caution), and alert-error (red, for problems). Just put your message text inside the div and you're done.

Tabs

Tabs work like the tabs on a filing cabinet — click one and it becomes the "active" tab that stands out from the rest. Wrap your tab buttons in a .tabs container, and add .tab-active to whichever one is currently selected. The active tab gets a colored underline so you can clearly see where you are. Switching tabs is up to you (a few lines of JavaScript), but the styling is all CSS.

Toggles

Toggle switches are those satisfying little on/off sliders you see on phones — slide right for on, slide left for off. In grit.css, they're built on regular <input type="checkbox"> elements, so they work natively without any JavaScript at all. Just add the .toggle class to a checkbox and it transforms from a boring square checkbox into a sleek rounded switch. Wrap it in a <label> with some text to describe what it controls.

Dropdown

Dropdowns are menus that pop open when you click a button — like a little drawer that slides out. The coolest part? They use the native HTML <details> and <summary> elements, so they work without a single line of JavaScript. The browser handles the open/close logic for you. Add .dropdown to the details element, style the trigger with .btn, and put your menu links inside a .dropdown-menu div.

Modal

Modals are those pop-up windows that appear on top of everything else and dim the background — like when an app asks "Are you sure you want to delete this?" They use the native HTML <dialog> element, so the browser handles accessibility and focus trapping for you. Open one with JavaScript (element.showModal()) and close it with element.close(). grit.css adds a nice backdrop blur, centered positioning, and smooth styling automatically.

Progress

Progress bars show how far along something is — like a loading bar in a video game. They're just a .progress container with a .progress-bar inside, and you control the fill width with an inline style="width: 45%". You can color them with variants like .progress-bar-success (green), .progress-bar-warning (yellow), and .progress-bar-error (red) to show different states.

Avatars

Avatars are circular profile pictures — the little round images you see next to usernames in chat apps or social media. Apply the .avatar class to an <img> tag and it'll be clipped into a perfect circle. They come in four sizes: .avatar-sm (small, 2rem), default (2.5rem), .avatar-lg (large, 3.5rem), and .avatar-xl (extra large, 5rem).

Tooltips

Tooltips are the little text bubbles that appear when you hover over something — like a sticky note that pops up to explain what a button does. They're 100% pure CSS, no JavaScript whatsoever. Just add a data-tooltip="Your message here" attribute to any element and the tooltip appears on hover. They work on buttons, links, plain text — anything!

Inputs

These are your standard form fields — text inputs, dropdown selects, and multi-line textareas. grit.css styles them all consistently with nice borders, proper spacing, and a satisfying focus ring (that glowing outline when you click into a field). Just use regular HTML form elements and they'll look great. Wrap each one in a .col.gap-xs div with a <label> above it for the cleanest layout.

Input Groups

Input groups let you attach extra stuff to the sides of an input field — like an "@" symbol before a username input, or a search icon at the end. Wrap the input and the addon in an .input-group div, and use .input-group-text for the addon element. They'll visually merge together into one seamless unit, like puzzle pieces snapping together.

Container

The container is like guardrails for your content — it centers everything on the page and caps the width at 1200px so your text doesn't stretch all the way across a giant monitor (which would make it really hard to read). It also adds a little horizontal padding on the sides so nothing bumps right up against the edge of the screen. Just wrap your content in a <div class="container"> and you're set.

Grid

The grid system lets you arrange things in columns — like a spreadsheet or a photo gallery. Use .grid-2, .grid-3, or .grid-4 for fixed column counts, or .grid-auto for a responsive layout that figures out the best number of columns automatically. On mobile, columns stack vertically so nothing gets squished. You can also use .span-2 or .span-full to make an item stretch across multiple columns.

Flex Utilities

Flex utilities are helper classes for arranging things in a line — horizontally with .row or vertically with .col. Think of them as instructions for how to line up your toys on a shelf. .row.between pushes items to the edges with space between them, .row.center groups everything in the middle, .wrap lets items flow onto the next line when they run out of room, and .grow makes an item stretch to fill any leftover space.

Spacing

Spacing utilities control the gaps and padding in your layout. Gap classes (.gap-xs through .gap-xl) set the space between items in a flex or grid container — like adjusting how far apart the books sit on a shelf. Padding classes (.pad-sm through .pad-xl) add breathing room inside an element — like the margins on a piece of paper so the text doesn't start right at the edge.

Gap (space between children)

Add a gap class to any flex or grid container. Gap only affects the space between children — it doesn't add space on the outer edges. If no gap class is set, .row, .col, and .grid default to 0.75rem.
